# 自媒体口播视频自动剪辑工具 — 设计文档

## 概述

一个本地运行的 Web 工具，帮助自媒体创作者自动剪辑口播视频：
自动去除语气词、空白停顿、重复段落，最终输出快节奏的 2-3 分钟成品视频。

## 使用场景

用户用手机录制多段口播视频。录错时当场重录，因此素材中包含：
- 重复的段落（不同 take 的同一段内容）
- 中文语气词（嗯、啊、那个、就是说 等）
- 过长的空白停顿

用户将多段原始视频上传到工具，自动检测并标记待删除片段，在界面上确认/调整后，导出单个成品视频。

## 架构

```
[浏览器前端]  ←→  [FastAPI 后端]  ←→  [Whisper]  ←→  [FFmpeg]
 单页 HTML         Python            语音识别        视频裁剪拼接
```

三条核心流水线：

| 阶段 | 做什么 | 工具 |
|------|--------|------|
| 语音识别 | 视频转逐词/逐句 + 时间戳文本 | faster-whisper (large-v3, 中文) |
| 智能检测 | 从文本中标记语气词、空白停顿、重复段落 | Python 逻辑 |
| 执行裁剪 | 按保留区间拼接，导出成品 | FFmpeg |

处理完成后输出新文件，不覆盖原始素材。

## 检测逻辑

### 语气词检测

预定义中文语气词列表，Whisper 逐词输出后匹配：

```
独立语气词（整段删除）：嗯、呃、啊、哦、啧、就是、那个、就是说、然后呢、对吧、
                     说白了、其实吧、你懂吧、怎么样、这个、那个
句子中的语气词（仅切该词时间段）
```

用户可自行增删语气词列表。

### 空白停顿检测

词间间隔超过阈值（默认 0.8 秒）即标记。连续沉默过长处整体删除。阈值用户可调。

### 重复段落检测

利用录音习惯（录错→当场重录，相邻内容相似度高）：

- Whisper 分句输出
- 相邻句文本相似度计算（SequenceMatcher / 编辑距离）
- 相似度 > 70% → 认为是同一内容的不同 take
- **保留后一句**（后录的通常是正确的），删除前一句

### 合并裁剪

三种检测各自产生"删除区间"。合并重叠区间后取反 = "保留区间"，传给 FFmpeg `concat` 拼接。

## 前端界面

单页 HTML，列表视图为主：

### 上传区

- 拖拽或点击上传多段 .mp4/.mov 文件
- 上传后以列表显示，左侧有 `≡` 拖拽手柄，可排序
- 每行显示文件名、时长，可预览播放，可删除
- "开始分析"按钮按当前顺序拼接后执行分析

### 确认编辑区（分析完成后展示）

- 每段一行，显示：序号、时间段、文本内容
- 绿色 = 保留，红色 = 标记删除
- 标记删除的行显示原因（"与段N 82% 相似" / "语气词" / "空白停顿 1.2s"）
- 右侧开关一键切换保留/删除
- 点击行播放对应片段
- 底部统计：保留时长 / 删除时长

### 导出

- "导出视频"按钮 → 后台 FFmpeg 拼接保留片段 → 浏览器下载

## 技术选型

- **后端**: Python FastAPI
- **语音识别**: faster-whisper, large-v3 中文模型
- **视频处理**: FFmpeg (concat protocol)
- **前端**: 单页 HTML + vanilla JS / 轻量框架
- **计算**: 全部本地，不上传云端

## 数据流

```
上传文件 → /api/upload 保存到临时目录
  → 用户排序确认 → /api/analyze
  → FFmpeg concat 临时拼接
  → faster-whisper 转写 (逐词/分句 + 时间戳)
  → 语气词/空白/重复检测
  → 返回检测结果 JSON（每段：起止时间、文本、建议操作、原因）
  → 前端渲染确认编辑界面
  → 用户确认 → /api/export
  → FFmpeg 按保留区间拼接 → 返回成品文件
```
